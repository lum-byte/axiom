The Core Insight
The vacuum doesn't walk.
The human navigates.
The layers don't coordinate.
They subscribe.

Every existing crawler is a monolith. It fetches, parses, decides, follows links, respects robots.txt, throttles itself, extracts content — all in one tightly coupled system. When one part is slow, everything is slow. When one part fails, the whole thing fails. The intelligence and the execution are the same process, which means neither can be optimal.

AXIOM's crawler is different in kind, not degree. The crawler is a bus — a raw acquisition spine that emits events. Every intelligent layer subscribes to those events and acts on its own signal independently. The preparser listens for robots.txt fetches and builds DomainMap. The WLM listens for domain topology events and updates traversal policy. The WLP listens for clean signal events and revises zone maps. The index daemon listens for everything and runs gradient steps. None of them talk to each other. They all talk to the bus.

The crawler does not understand what it fetches. It does not know what robots.txt means. It does not know what a topology class is. It fetches bytes and emits events. That is the complete job description. Everything else is someone else's responsibility.

Alpine — Doubly Justified
The same container.
Two completely different jobs.
Zero additional infrastructure.

The signal_kernel's Alpine container was built for the critical path — stripping noise from pages at query time before the LLM sees them. That justification stands alone and is already proven: 1.1MB of Stripe docs to 8,460 bytes in 7.4ms. 99.27% reduction.

But the layered crawler architecture reveals a second justification that is equally load-bearing. The Alpine container is the strip layer of the offline crawler. Every byte the crawler fetches — from Wikipedia, from arXiv, from any crawled domain — passes through the same warm grep pipeline before any Python intelligence layer receives it. The WLM never sees raw HTML. The WLP never sees raw HTML. The index daemon never sees raw HTML. No intelligence layer ever processes noise.

Critical Path (query time)
Online use of Alpine
AXIOM receives a query. Phantom fetches the target page. Raw HTML pipes to the warm Alpine container. grep recipe runs. Clean signal exits. Haiku sees only signal. 7.4ms on Stripe docs. This is the proven use case. 99%+ reduction at C speed.

Container is already warm from the previous query. Zero spawn cost on consecutive invocations. The warm container optimization from pipeline.py earns its keep here.
Offline Path (crawl time)
Crawler use of Alpine
The crawler fetches a Wikipedia article. Same warm Alpine container. Same grep recipe for WIKIPEDIA_ARTICLE topology. Clean signal exits. Index daemon receives clean signal. Intent clusterer receives clean signal. Topology encoder receives clean signal.

The offline batch job runs at the same 99%+ noise reduction rate as the critical path. structural_layer.pt is trained on clean signal, not raw HTML. The MFT has never seen noise. Its weights encode pure structure.
◈
This is why Alpine is not an implementation detail. It is load-bearing infrastructure for both the online and offline paths simultaneously. One container. One image. One set of grep recipes. Two completely different consumers. The warm container that the crawler batch-processes with at 80,000 links per second is the same warm container that serves the next user query in 7.4ms. There is no additional infrastructure cost for the offline path. Alpine was already there.
The Preparser Layer
robots.txt is not a rule.
It is a structural map.

Every other crawler reads robots.txt to know what not to fetch. AXIOM's preparser reads robots.txt to understand the domain's topology. Every directive is a structural signal. Every disallowed path encodes a topology class. Every crawl delay encodes a rate limit profile. Every sitemap URL is a free gift — the domain's complete topology handed over without a single content fetch. 

# What a traditional crawler reads:
#   "Disallow: /admin" → skip /admin
#
# What AXIOM's preparser reads:

User-agent: *
Disallow: /admin/          → AUTH_REDIRECT topology
                             high bot_detection_probability
                             zero signal probability
                             TraversalPolicy: skip entirely

Disallow: /search          → DYNAMIC_SEARCH_RESULTS topology
                             no stable signal zone
                             skip — unreliable extraction

Disallow: /checkout/       → ECOMMERCE_CHECKOUT topology
                             signal exists but auth-gated
                             FrictionForecast: auth = 0.97

Disallow: /api/internal/   → API_INTERNAL topology
                             not public signal — skip

Crawl-delay: 10            → rate_limit_profile
                             10.0 req/s ceiling for this domain
                             encoded into TraversalPolicy
                             never triggers a 429 reactively

Sitemap: /sitemap.xml      → GOLD — full topology map
                             parse sitemap → complete URL frontier
                             path patterns → topology classes
                             /blog/* → NEWS_ARTICLE
                             /docs/* → SAAS_DOCS
                             /api/*  → REST_API_JSON
                             entire domain classified from one request
@dataclass
class DomainMap:
    domain: str

    # derived from robots.txt
    disallowed_topology_classes: Dict[str, str]   # path → topology class
    allowed_signal_paths:        List[PathPattern] # where signal lives
    crawl_delay_seconds:         float             # rate limit profile
    sitemap_urls:                List[str]          # process for full frontier

    # derived from sitemap + structural analysis
    path_topology_map:           Dict[str, str]    # /blog/* → NEWS_ARTICLE
    friction_zones:              List[PathPattern] # high friction paths
    signal_zones:                List[PathPattern] # high signal density paths
    bot_mitigation:              str               # cloudflare / none / custom
    render_requirements:         Dict[str, str]    # path → static/headless
    rate_limit_profile:          RateLimitProfile

    # what gets handed to the crawler
    crawl_manifest:              CrawlManifest     # ordered URLs + topology hints
                                                     # crawler executes this blindly
                                                     # no decisions required
The Sitemap Gift
Full domain topology from one request
A domain that publishes /sitemap.xml hands AXIOM its complete URL topology before the crawler fetches a single content page. Every URL, often with lastmod timestamps and changefreq hints. The preparser classifies every path pattern and generates a fully typed CrawlManifest in one HTTP request.
The Disallow Signal
Friction topology without touching it
Every Disallow directive tells AXIOM what kind of friction lives at that path — without the crawler ever triggering a 403 or a bot challenge. The preparser encodes the friction type into the DomainMap. The WLM's FrictionForecast is pre-populated before the first content fetch. The system knows the friction landscape of a domain from its robots.txt alone.
The Crawl-Delay Encoding
Rate limiting that never reacts
Traditional crawlers discover rate limits by triggering 429s and backing off. AXIOM reads Crawl-delay, encodes it as a RateLimitProfile, and hands the crawler a pacing manifest. The crawler never gets rate-limited because the preparser already knew the ceiling. A 429 in AXIOM's logs is an architecture bug, not a normal operating condition.
The Event Bus
Events the crawler emits.
Layers that subscribe.

The crawler emits typed events. Every layer subscribes to exactly the events it cares about. No layer calls another layer. No layer waits for another layer. The event bus is the only shared interface. Each layer processes its subscribed events independently, on its own schedule, at its own priority level. 

# Events emitted by the crawler bus

@dataclass(frozen=True)
class RawFetchEvent:          # Layer 1 emits this
    url:            str
    raw_bytes:      bytes
    status_code:    int
    headers:        Dict[str, str]
    fetch_latency:  float
    is_robots_txt:  bool       # preparser subscribes when True
    is_sitemap:     bool       # preparser subscribes when True
    run_id:         str

@dataclass(frozen=True)
class CleanSignalEvent:        # Layer 2 (Alpine) emits this
    url:                str
    clean_signal:       str
    topology_class:     str
    token_reduction:    float
    signal_density:     float
    extraction_empty:   bool
    run_id:             str
    # WLM, WLP, index_daemon, ingestion all subscribe to this

@dataclass(frozen=True)
class DomainTopologyEvent:     # preparser emits after building DomainMap
    domain:             str
    domain_map:         DomainMap
    # WLM subscribes to update TraversalPolicy
    # index_daemon subscribes to update structural_layer.pt

@dataclass(frozen=True)
class CrawlManifestReadyEvent: # preparser emits — crawler subscribes
    domain:             str
    manifest:           CrawlManifest
    # crawler receives this and begins fetching
    # crawler does not generate its own manifest

@dataclass(frozen=True)
class ZoneMapUpdatedEvent:     # WLP emits after zone map revision
    topology_class:     str
    new_zone_map:       ZoneMap
    # topology_parser.py subscribes — triggers recipe recompilation

@dataclass(frozen=True)
class SurpriseEvent:           # surprise_detector.py emits
    topology_class:     str
    surprise_score:     float
    dissolve_triggered: bool
    # index_daemon subscribes — decides gradient step vs dissolve
◈
The key rule: layers subscribe to events from the bus. They never call each other directly. The WLM does not call the preparser to ask for a DomainMap. It subscribes to DomainTopologyEvent and processes it when it arrives. The index daemon does not poll for surprise events. It subscribes and reacts. This decoupling means any layer can fail independently without affecting the others. The crawler keeps fetching. Alpine keeps stripping. The bus keeps emitting.

The Complete Picture
What AXIOM actually is.
Stated plainly.
01
A crawler that does not think
crawler/fetcher.py receives a list of URLs and fetches them. It knows nothing about topology, robots.txt, rate limits, or signal. That knowledge lives in the preparser's CrawlManifest. The crawler executes the manifest blindly and emits events. This is the vacuum. Its hyperefficiency comes from doing exactly one thing.
02
An Alpine container that strips everything, always
Every byte fetched — online or offline, query-time or crawl-time — passes through the same warm grep pipeline before any intelligence layer sees it. The WLM, WLP, index daemon, and ingestion layer have never seen raw HTML. They have only ever seen clean signal. The MFT is trained on pure structure. This is what 99.27% reduction means at architectural scale.
03
A preparser that reads the web's own documentation
robots.txt and sitemaps are the web's self-description. Every domain publishes exactly where it wants to be crawled, at what pace, and what topology its paths belong to. The preparser reads this documentation and converts it into structural knowledge. The crawler then executes exactly what the domain asked for. No 429s. No bot challenges. No reactive backoff. The preparser knew the rules before the first request.
04
A world model that learns in parallel, never blocking
WLM and WLP run parallel, subscribing to the same bus, acting on different signals. Neither waits for the other. Neither calls the other. Both contribute to the MFT through the index daemon, which receives their outputs as events and runs gradient steps asynchronously. A query in progress never waits for a learning update. A learning update never waits for a query to complete.
05
An index that grows every idle second, forever
When AXIOM is not serving queries, the crawler runs at full 5080 capacity. Every idle second is more of the web encoded into weights. Every domain the crawler maps is a domain that future queries hit with zero live traversal cost. The moat widens automatically. No code changes. No scheduled jobs. No human intervention. The daemon never terminates. The loop never stops.

The architecture is justified at every level. Alpine is justified twice — online and offline. The preparser's intelligence justifies the crawler's simplicity. The crawler's simplicity justifies the preparser's authority. The event bus justifies the independence of every layer. Every design decision supports every other design decision. This is what a coherent architecture looks like.
