signal_kernel — API Reference

Complete reference for pipeline.py, the public entry point into the grep extraction engine. This page covers every function signature, every contract type accepted and returned, and the exact failure modes you must handle as a caller.

Version: This reference covers signal_kernel v0.4.x. The v0.3.x pipeline interface is deprecated and will be removed in v0.5.0. See the changelog for migration details.

Overview

pipeline.py manages the full lifecycle of one grep extraction run. It is stateless between invocations — every call receives a fresh KernelInput and returns a KernelOutput. No shared mutable state exists between concurrent pipeline executions.

The pipeline does three things in sequence:

  1. Validates the recipe via validator.py and constructs a RecipeMount — the proof that the recipe is safe to execute.
  2. Spawns an Alpine container subprocess, writes raw_content to stdin, and collects stdout as clean_signal.
  3. Builds KernelOutput, emits PipelineTelemetry to Witness, and fires a FeedbackEvent to feedback.py.

Steps 2 and 3 are wrapped in a timeout. If the subprocess exceeds DEFAULT_SUBPROCESS_TIMEOUT_MS (5000ms), the pipeline kills it, constructs an empty KernelOutput via make_empty_kernel_output(), and returns gracefully. The AXIOM graph continues. Kernel timeout is never a hard stop.

Installation

signal_kernel requires Python 3.11+ and Docker (for the Alpine subprocess). Install into an isolated virtualenv. 

python3 -m venv .venv
source .venv/bin/activate
pip install signal-kernel==0.4.2 --break-system-packages

Verify the installation by running the built-in smoke test. This test spawns a real Alpine container and runs news_article.sh against a fixture page. If Docker is not running, the smoke test fails with ContainerSpawnError. 

python3 -m signal_kernel.smoke_test
# Expected output:
# [PASS] KernelInput construction
# [PASS] RecipeMount: news_article.sh (sha256 verified)
# [PASS] Pipeline execution: 487 bytes, 70.2% reduction, 312ms
# [PASS] FeedbackEvent emitted
# [PASS] PipelineTelemetry emitted to Witness

If you see RecipeManifestCorruption during the smoke test, the hardcoded recipe manifest was not included in the package. Reinstall from the internal registry, not from PyPI.

Quick Start

The minimal invocation: construct a KernelInput, call pipeline.run(), check extraction_empty. 

from signal_kernel.contracts import (
    KernelInput,
    KernelOutput,
    new_run_id,
    TopologyClassStr,
)
from signal_kernel import pipeline

# Build the input contract
kernel_input = KernelInput(
    raw_content=html_string,          # UTF-8 HTML, max 4MB
    topology_class=TopologyClassStr("NEWS_ARTICLE"),
    intent_vector_hash=intent_hash,   # SHA-256 hex digest
    content_type="html",
    source_url="https://example.com/article",
    run_id=new_run_id(),              # always use new_run_id()
)

# Execute the pipeline
result: KernelOutput = pipeline.run(kernel_input)

# Check the result
if result.extraction_empty:
    # Not always an error — could be paywall, JS-only page, etc.
    # feedback.py tracks the rolling empty rate for this topology class
    handle_empty_extraction(result)
else:
    send_to_llm(result.clean_signal)
    print(f"Reduction: {result.token_reduction_pct:.1%}")
    print(f"Signal density: {result.signal_density:.3f}")

pipeline.run() never raises for extraction failures. The only exceptions that propagate to the caller are RecipeMountError and RecipeInjectionAttempt — both require human review and cannot be retried automatically.

pipeline.run(input: KernelInput) → KernelOutput

The single public function in pipeline.py. Executes one complete extraction cycle: recipe validation → subprocess spawn → content extraction → telemetry emission.

Parameters

Parameter Type Description
input KernelInput The fully validated input contract. Constructing KernelInput validates all fields — a KernelInput that exists has already passed its invariants.

Returns

A KernelOutput in one of two states:

  • Successful extraction: extraction_empty=False, clean_signal contains the extracted text, clean_byte_count > 0.
  • Empty extraction: extraction_empty=True, clean_signal="", clean_byte_count=0. Constructed via make_empty_kernel_output(). Returned on timeout, spawn failure, zero-byte output, or stdin encoding error.

Exceptions raised to caller

Only two exceptions propagate past pipeline.run(). All others are caught internally and result in an empty KernelOutput.

Exception When Action required
RecipeMountError Recipe file missing, hash mismatch, injection detected, structural validation failed, or dry-run failure. Hard stop. Do not retry. The recipe for this topology class is unavailable. Fall back to GENERIC_HTML if available. File an alert.
RecipeInjectionAttempt validator.py detected a shell injection pattern in a compiler-generated recipe. Hard stop. Do not retry under any circumstances. Examine the InjectionAuditRecord in the Witness stream. Requires forensic review of the recipe compiler output.

KernelInput

Everything the kernel needs to execute a single extraction. Frozen dataclass — immutable after construction. Construction validates all fields immediately. A KernelInput that exists has already passed every invariant check. 

@dataclass(frozen=True)
class KernelInput:
    raw_content:        str             # UTF-8 HTML or JSON, max 4MB
    topology_class:     TopologyClassStr  # matches [A-Z][A-Z0-9_]{1,63}
    intent_vector_hash: IntentVectorHash  # SHA-256 hex digest, 64 chars
    content_type:       ContentType     # "html" | "json"
    source_url:         SourceURL       # must begin with http:// or https://
    run_id:             RunID           # canonical UUID4, use new_run_id()
    received_at:        datetime        # auto-set to UTC now

Validation rules

  • raw_content must be non-empty and ≤ MAX_RAW_CONTENT_BYTES (4 194 304 bytes) when UTF-8 encoded. Pages exceeding this limit indicate a fetcher misconfiguration upstream — the kernel should not receive them.
  • topology_class must match [A-Z][A-Z0-9_]{1,63}. This is enforced by a compiled regex. Lowercase or hyphenated class names are rejected.
  • intent_vector_hash must be a 64-character lowercase hexadecimal SHA-256 digest. Mixed-case or truncated hashes are rejected with a descriptive error naming the field.
  • run_id must be generated via new_run_id(). Never construct run IDs manually. UUID4 guarantees cryptographic randomness — sequential IDs compromise audit trail integrity.

Properties

input.raw_byte_count     # int: UTF-8 encoded byte count of raw_content
input.is_html            # bool: content_type == "html"
input.is_json            # bool: content_type == "json"
input.is_known_topology  # bool: topology_class in KNOWN_TOPOLOGY_CLASSES
input.is_hardcoded_topology  # bool: hardcoded recipe exists for class

Construction errors

KernelInput.__post_init__ raises ValueError for any constraint violation. pipeline.py wraps these into typed subclasses of KernelInputError with execution context attached. 

# This raises ValueError: "raw_content must not be empty"
bad_input = KernelInput(
    raw_content="",
    topology_class=TopologyClassStr("NEWS_ARTICLE"),
    intent_vector_hash="a" * 64,
    content_type="html",
    source_url="https://example.com",
    run_id=new_run_id(),
)

# This raises ValueError: topology_class format invalid
bad_input = KernelInput(
    raw_content="content",
    topology_class=TopologyClassStr("news-article"),  # hyphens not allowed
    intent_vector_hash="a" * 64,
    content_type="html",
    source_url="https://example.com",
    run_id=new_run_id(),
)

KernelOutput

The result of one grep pipeline execution. Frozen dataclass. extraction_empty=True is a valid, non-error state — it means the page had no signal in the expected structural zone, not that the kernel failed. 

@dataclass(frozen=True)
class KernelOutput:
    clean_signal:         str             # extracted text (empty if extraction_empty)
    raw_byte_count:       int             # bytes of raw_content
    clean_byte_count:     int             # bytes of clean_signal
    token_delta_estimate: int             # estimated tokens removed (chars/4)
    recipe_used:          str             # path of the recipe that ran
    topology_class:       TopologyClassStr
    extraction_empty:     bool            # True if grep produced zero bytes
    latency_ms:           float           # wall-clock time for the full run
    run_id:               RunID
    produced_at:          datetime        # auto-set to UTC now

Properties

output.compression_ratio      # float: clean_bytes / raw_bytes
output.token_reduction_pct    # float: 1.0 - compression_ratio
output.signal_density         # float: non-whitespace chars / total chars
output.is_under_stripping     # bool: token_reduction_pct < 0.40
output.is_over_stripping      # bool: token_reduction_pct > 0.95
output.is_in_target_range     # bool: 0.60 <= token_reduction_pct <= 0.80

is_in_target_range is the production quality indicator. A recipe performing in the 60–80% token reduction range is operating correctly. Below 40% indicates under-stripping (recipe not aggressive enough). Above 95% indicates over-stripping (recipe discarding signal content).

Consuming the output

result = pipeline.run(kernel_input)

# Always check extraction_empty first
if not result.extraction_empty:
    print(result.clean_signal)
    print(f"Token reduction: {result.token_reduction_pct:.1%}")
    print(f"Signal density:  {result.signal_density:.3f}")
    print(f"In target range: {result.is_in_target_range}")
    print(f"Latency:         {result.latency_ms:.0f}ms")
else:
    # Pass to feedback.py for quality tracking — empty is valid signal
    quality = make_quality_from_output(result)
    print(f"Empty extraction. Quality score: {quality.composite_score:.4f}")

The dependency direction is strict: everything imports from contracts.py — contracts.py imports nothing from the kernel. If you find an import from pipeline.py or feedback.py inside contracts.py, that is a dependency inversion and must be corrected before merging.

Topology Classes

A topology class is a string identifier that maps a page to its structural template and the recipe that extracts signal from it. The five hardcoded topology classes have hand-written, hash-verified recipes that are immutable at runtime.

Class Recipe Signal zone Notes
NEWS_ARTICLE news_article.sh <article> Hardcoded. Strips nav, aside, footer, script, style.
SAAS_DOCS saas_docs.sh <main> Hardcoded. Preserves code block indentation.
REST_API_JSON rest_api_json.sh data/results envelope Hardcoded. Depth-aware JSON key filtering.
JSON_LD_STRUCTURED json_ld.sh <head> script blocks Hardcoded. Head-only extraction, body LD+JSON discarded.
ECOMMERCE_PRODUCT ecommerce.sh data-product attributes Hardcoded. Attribute-level noise erasure.
NEWS_ARTICLE_PAYWALLED → NEWS_ARTICLE <article> Not hardcoded. Falls back to NEWS_ARTICLE via PARENT_CLASS_MAP. Expected empty extraction rate is higher — paywall login walls produce zero signal.
GENERIC_HTML generic.sh <body> Compiler-generated fallback. Used when no specific recipe exists for the detected topology class. Lower signal density expected — body extraction is undiscriminating.

Quality Thresholds

All thresholds are defined as constants in contracts.py. feedback.py and topology_parser.py import them from there — they are never duplicated. 

from signal_kernel.contracts import (
    SIGNAL_DENSITY_FLOOR,           # 0.35 — below: over-stripping or no signal
    SIGNAL_DENSITY_CEILING,         # 0.95 — above: likely retaining markup
    MIN_MEANINGFUL_TOKEN_REDUCTION, # 0.40 — below: recipe under-stripping
    MAX_MEANINGFUL_TOKEN_REDUCTION, # 0.95 — above: recipe over-stripping
    EMPTY_EXTRACTION_RATE_THRESHOLD, # 0.25 — triggers recompilation recommendation
    QUALITY_WINDOW_SIZE,            # 100  — rolling window samples per topology class
)

The production target range is 60–80% token reduction. Recipes consistently outside this range are flagged for recompilation via FeedbackEvent.recompilation_recommended=True. The topology_parser.py learning loop acts on these signals — feedback.py only signals, it does not invoke the compiler.

ExtractionQuality composite score

ExtractionQuality.composite_score is a human-readable quality metric in [0.0, 1.0]. It is computed from signal_density and token_reduction_pct: 

@property
def composite_score(self) -> float:
    if self.empty_extraction:
        return 0.0
    density_score = min(self.signal_density / SIGNAL_DENSITY_FLOOR, 1.0)
    r = self.token_reduction_pct
    if r < MIN_MEANINGFUL_TOKEN_REDUCTION:
        reduction_score = r / MIN_MEANINGFUL_TOKEN_REDUCTION
    elif r > MAX_MEANINGFUL_TOKEN_REDUCTION:
        over = r - MAX_MEANINGFUL_TOKEN_REDUCTION
        reduction_score = max(0.0, 1.0 - over * 10.0)
    else:
        reduction_score = 1.0
    return round((density_score + reduction_score) / 2.0, 4)

This is a diagnostic metric, not a training loss. topology_parser.py uses token_reduction_pct and signal_density directly for its decisions — it does not use composite_score.

Exception Handling

All kernel exceptions inherit from KernelException. Every exception carries a structured to_audit_dict() method and an exception_code string for log filtering. The is_hard_stop attribute tells you whether pipeline.run() re-raises or degrades gracefully. 

from signal_kernel.exceptions import (
    KernelException,
    RecipeMountError,
    RecipeInjectionAttempt,
    is_hard_stop,
    classify,
)

try:
    result = pipeline.run(kernel_input)
except RecipeInjectionAttempt as exc:
    # Forensic stop — full recipe content is in the InjectionAuditRecord
    alert_security_team(exc.to_audit_dict())
    raise SystemExit(1)
except RecipeMountError as exc:
    # Hard stop — recipe unavailable for this topology class
    logger.critical("Recipe mount failed", extra=exc.to_audit_dict())
    result = make_empty_kernel_output(
        run_id=kernel_input.run_id,
        topology_class=kernel_input.topology_class,
        recipe_used="",
        raw_byte_count=kernel_input.raw_byte_count,
        latency_ms=0.0,
    )
except KernelException as exc:
    # Should never reach here from pipeline.run() — all other exceptions
    # are caught internally. If this fires, it is a programming error.
    logger.critical("Unexpected KernelException escaped pipeline.run()",
                    extra=classify(exc))
    raise

Exception codes

Every exception carries a stable exception_code string for log filtering without matching on class names. Use EXCEPTION_REGISTRY to reconstruct exception types from serialized audit records. 

from signal_kernel.exceptions import EXCEPTION_REGISTRY, exception_from_code

# Look up exception class from a code string found in a log record
exc_class = exception_from_code("KERNEL_RECIPE_INJECTION")
# → RecipeInjectionAttempt

exc_class = exception_from_code("KERNEL_OUTPUT_TIMEOUT")
# → SubprocessTimeout

Checkpoint System

The checkpoint system is Process 2 — entirely separate from the grep pipeline (Process 1). crond fires mft_checkpoint.sh every 15 minutes. The four store files are archived to /store/checkpoints/ as .tar.gz. Up to 48 archives are retained (12 hours at 15-minute intervals).

An unhealthy checkpoint system does not stop Process 1. It triggers a Witness alert. The extraction pipeline continues regardless of whether crond is alive.

Restore on startup

On TAG startup, checkpoint_monitor.py checks whether all four store files are present and valid. If any are missing or corrupt, restore.sh is invoked. It iterates archives newest-to-oldest, verifying each with tar -tzf before extracting. If all 48 archives are corrupt, CheckpointExhaustionError is raised and TAG cannot start. 

from signal_kernel.exceptions import (
    RestoreFailure,
    CheckpointExhaustionError,
    RestorePartialError,
)

# In TAG startup sequence:
try:
    checkpoint_monitor.restore_if_needed()
except CheckpointExhaustionError as exc:
    # All 48 archives are corrupt — 12-hour window unrecoverable
    logger.critical("All checkpoints corrupt", extra=exc.to_audit_dict())
    raise SystemExit("TAG cannot start: no valid checkpoint found")
except RestorePartialError as exc:
    # Archive found but not all 4 store files were restored
    logger.critical("Partial restore", extra=exc.to_audit_dict())
    raise
except RestoreFailure as exc:
    logger.critical("Restore failed", extra=exc.to_audit_dict())
    raise

PipelineTelemetry

PipelineTelemetry is emitted by pipeline.py after every invocation — successful or empty. It is not optional. Every field is either directly measured or deterministically derived. No inference. No estimation beyond token_delta_estimate (characters divided by 4, the standard LLM token approximation). 

from signal_kernel.contracts import make_pipeline_telemetry

# pipeline.py constructs this after every run — callers do not build it
telemetry = make_pipeline_telemetry(
    output=kernel_output,
    lifecycle=container_lifecycle,
    is_hardcoded_recipe=recipe_mount.is_hardcoded,
)

# Emit to Witness as a structured log record
logger.info("pipeline.telemetry", extra=telemetry.to_log_dict())

Daemon Interface

The AXIOM controller communicates with the TAG daemon via interface.py. The controller sends a DaemonQuery and receives a DaemonResponse. The kernel does not exist at the graph level — the graph sees only query in, DaemonResponse out. 

from signal_kernel.contracts import DaemonQuery, DaemonResponse, new_run_id

# AXIOM controller builds this
query = DaemonQuery(
    intent_vector_hash=IntentVectorHash("a" * 64),
    target_urls=(
        "https://example.com/article-1",
        "https://example.com/article-2",
    ),
)

# TAG daemon returns this
response: DaemonResponse = interface.handle(query)

# The response contains everything needed to execute the pipeline
print(response.topology_class)        # "NEWS_ARTICLE"
print(response.phase)                 # "knows" | "predicts" | "learns"
print(response.is_phase_three)        # True if phase == "knows"
print(response.highest_friction_url)  # URL predicted hardest to extract
print(response.mft_hit)               # True if a prior run matched

Phases

Phase Meaning Recipe source
learns TAG has seen this topology class fewer than 10 times. Observations are being collected. No compiled policy yet. Fallback: GENERIC_HTML
predicts TAG has compiled an initial recipe but is still gathering quality signal. Recompilation may occur after this run. Compiler-generated recipe
knows TAG has a stable, high-quality recipe backed by sufficient window samples. Direct policy execution. Compiler-generated or hardcoded recipe

Security Model

The kernel's security boundary is validator.py. Every compiler-generated recipe passes through injection detection before it may execute. The injection patterns are defined in contracts.py as INJECTION_PATTERNS — a tuple of 18 regex patterns that validator.py tests against every recipe before permitting execution. 

from signal_kernel.contracts import (
    INJECTION_PATTERNS,        # tuple of 18 regex patterns
    ALLOWED_RECIPE_COMMANDS,   # frozenset: grep sed awk cat cut tr head tail sort uniq
    MAX_RECIPE_LINE_COUNT,     # int: 50
)

Hardcoded recipes are never injection-tested — they are human-authored and hash-verified. If a hardcoded recipe hash does not match its manifest entry, RecipeHashMismatch is raised with is_tamper_event=True. This is treated as a filesystem tamper event, not a validation failure.

Injection detection fires on the compiler-generated recipe before the recipe is written to disk. If detection fires:

  1. Capture the full recipe content in InjectionAuditRecord. Do not truncate. The full content is required for forensic analysis of how the recipe compiler produced a hostile payload.
  2. Raise RecipeInjectionAttempt.
  3. Never write the recipe to disk.
  4. Never pass it to the kernel subprocess under any circumstances.
  5. Alert Witness immediately via KernelAuditEvent with severity="critical".

The injection vector is adversarial web pages causing the recipe compiler to emit a shell injection payload. A hostile page can produce a DOM structure that the compiler translates into a $(command) or backtick execution in the generated grep pipeline. The injection would execute inside the Alpine container — which has no network access (network_mode: none) and runs as a non-root user, limiting the blast radius. But the kernel's defense-in-depth does not rely on container isolation — validator.py is the primary barrier.

Migrating from v0.3.x

The v0.4.0 release replaced the old KernelRunner class with the stateless pipeline.run() function. If you are upgrading from v0.3.x, three changes are required.

1. Replace KernelRunner with pipeline.run()

# v0.3.x — deprecated
from signal_kernel import KernelRunner
runner = KernelRunner(topology_class="NEWS_ARTICLE")
result = runner.extract(html, url=url)

# v0.4.x — current
from signal_kernel import pipeline
from signal_kernel.contracts import KernelInput, new_run_id
result = pipeline.run(KernelInput(
    raw_content=html,
    topology_class="NEWS_ARTICLE",
    intent_vector_hash=compute_hash(html),
    content_type="html",
    source_url=url,
    run_id=new_run_id(),
))

2. Update exception handling

# v0.3.x — deprecated exception names
from signal_kernel.errors import ExtractionError, TimeoutError

# v0.4.x — typed exception hierarchy
from signal_kernel.exceptions import (
    RecipeMountError,        # was: ExtractionError(reason="recipe")
    SubprocessTimeout,       # was: TimeoutError
    EmptyExtractionError,    # was: ExtractionError(reason="empty")
    RecipeInjectionAttempt,  # new in v0.4 — no v0.3 equivalent
)

3. Update quality metric access

# v0.3.x — deprecated attributes
result.reduction_pct   # renamed
result.density         # renamed

# v0.4.x — current attribute names
result.token_reduction_pct   # was: reduction_pct
result.signal_density        # was: density

Troubleshooting

ContainerSpawnError on every request

Docker daemon is not running or is under memory pressure. Check: 

docker info
docker stats --no-stream
# If OOM: increase Docker memory limit in Docker Desktop → Settings → Resources

High empty extraction rate for NEWS_ARTICLE

If window_empty_extraction_rate in FeedbackEvent exceeds EMPTY_EXTRACTION_RATE_THRESHOLD (0.25), the corpus for this topology class has shifted. Common causes:

  • Source URLs are returning login walls or paywalls — check source_url values against known paywall domains. These should be classified as NEWS_ARTICLE_PAYWALLED, not NEWS_ARTICLE.
  • The article DOM has changed — a publication redesigned their template and <article> is no longer the signal zone. topology_parser.py will detect this from the quality signal and recompile.
  • JavaScript-rendered content — Phantom is delivering the JS bundle instead of the rendered DOM. Check TraversalConfig.render_mode. Should be "headless" for JS-heavy news sites.

RecipeHashMismatch on startup

A hardcoded recipe file was modified after deployment. is_tamper_event=True on the exception means the file on disk does not match the hash committed to the manifest. This is treated as a security event, not a configuration error. 

# Verify all hardcoded recipe hashes from the shell
cd /app/recipes/hardcoded
for f in *.sh; do
    expected=$(grep "\"$f\"" manifest.json | grep -o '"hash":"[^"]*"' | cut -d'"' -f4)
    actual=$(sha256sum "$f" | cut -d' ' -f1)
    [ "$expected" = "$actual" ] && echo "OK: $f" || echo "MISMATCH: $f"
done

StdinEncodingError on specific URLs

The page contains bytes that cannot be encoded as UTF-8. Common with pages that declare charset=UTF-8 but embed ISO-8859-1 or Windows-1252 bytes in JavaScript strings or data attributes. 

from signal_kernel.exceptions import StdinEncodingError

# The audit dict tells you which URL and what encoding error fired
try:
    result = pipeline.run(kernel_input)
except StdinEncodingError as exc:
    # This should not escape pipeline.run() — log as a bug if it does
    logger.error("StdinEncodingError escaped", extra=exc.to_audit_dict())
